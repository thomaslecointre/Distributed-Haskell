Traitement pour l'esclave

Je dois mettre à jour le traitement, je m'en occupe après le poster aussi ;)

L'esclave prend en argument : [String] tel que ["ip","port","keyword","serie","#n_saison","#saison"]
ip = "www.exemple.com"
port = "8080"
keyword = "Ned"
serie = "Game of Thrones"
n_saison = "1"
#saison = "7"

La sortie est à définir : ex :
- tri chronogique renvoie : [n_saison, occurence_episode1, occ_ep2, ...] tel que [1,2,0,3,1]

- tri par saison renvoie : ex : [(n_saison,-1),(episodeC,occurrenceMax1),(episodeB,occurrenceMax2),(episodeD,occurrenceMax3),(episodeA,occurrenceMax4)] tel que [(1,-1),(3,3),(1,2),(4,1),(2,0)]

- tri par pertinence renvoie du json : {"1_3" : 4, "2_1" : 4, "2_3" : 4, "1_1" : 3, "1_2" : 3, "2_2" : 3}

- statistiques : crée un json {mot, occurence} par épisode




Dans le main, mettre un argument et faire un switch case sur le tri souhaité !

Les étapes :
LireArgumentEsclave : crée les url à traiter
LireURLJson : renvoie le résumé
GetWordsFilev2 : supprime les stop words
PorterStemmer : utilise l'algorithme de Porter

Statisticsv2 : pour faire le nuage de mots d'un épisode
KeyWordOccurrence : pour retourner l'occurence d'un keyword
Sort : pour trier et créer du json
